# Feature Specification: 발행 대시보드 새로고침 버튼 개선

**Feature Branch**: `001-refresh-button`
**Created**: 2026-01-16
**Status**: Draft
**Input**: Linear JEO-13 - 발행 대시보드에서 새로고침 버튼 개선하기

## 문제 정의

현재 발행 대시보드의 새로고침 버튼에 다음과 같은 UX 문제가 있습니다:

1. **위치 문제**: 새로고침 버튼과 Dialog 닫기(x) 버튼이 너무 가까이 있어 실수로 잘못된 버튼을 클릭할 수 있음
2. **직관성 부족**: 새로고침 버튼이 무엇을 의미하는지 직관적으로 알기 어려움
3. **크기 문제**: 새로고침 버튼이 너무 커서 다른 UI 요소와 조화를 이루지 않음

## User Scenarios & Testing

### User Story 1 - 새로고침 버튼 시각적 개선 (Priority: P1)

사용자가 발행 대시보드에서 새로고침 버튼을 쉽게 인식하고 실수 없이 클릭할 수 있습니다.

**Why this priority**: 잘못된 버튼 클릭으로 인한 사용자 실수를 방지하고, 기본적인 사용성을 개선하는 것이 가장 중요합니다.

**Independent Test**: 새로고침 버튼을 클릭하여 발행 목록이 갱신되는지 확인하고, 닫기 버튼과의 실수 클릭이 발생하지 않는지 테스트합니다.

**Acceptance Scenarios**:

1. **Given** 발행 대시보드가 열려 있을 때, **When** 사용자가 새로고침 버튼 영역을 보면, **Then** 새로고침 기능임을 즉시 인식할 수 있어야 합니다
2. **Given** 발행 대시보드가 열려 있을 때, **When** 사용자가 새로고침 버튼을 클릭하면, **Then** 발행 목록이 갱신되어야 합니다
3. **Given** 발행 대시보드가 열려 있을 때, **When** 사용자가 닫기(x) 버튼 근처를 클릭하면, **Then** 새로고침 버튼과 명확히 구분되어 의도한 버튼만 동작해야 합니다

---

### User Story 2 - 버튼 크기 및 레이아웃 조정 (Priority: P2)

새로고침 버튼의 크기가 다른 UI 요소와 일관되게 조정되어 전체적인 디자인 조화를 이룹니다.

**Why this priority**: 시각적 일관성은 사용자 경험을 향상시키지만, 기능적 개선보다는 우선순위가 낮습니다.

**Independent Test**: 버튼 크기가 Obsidian 디자인 가이드라인에 맞는지 시각적으로 확인합니다.

**Acceptance Scenarios**:

1. **Given** 발행 대시보드가 열려 있을 때, **When** 사용자가 버튼 영역을 보면, **Then** 새로고침 버튼이 다른 버튼들과 크기가 일관되어야 합니다
2. **Given** 다양한 화면 크기에서, **When** 발행 대시보드를 열면, **Then** 버튼 레이아웃이 적절하게 유지되어야 합니다

---

### Edge Cases

- 새로고침 중일 때 버튼을 다시 클릭하면 어떻게 되는가? → 중복 요청을 방지하고 로딩 상태를 표시합니다
- 네트워크 오류로 새로고침이 실패하면 어떻게 되는가? → 사용자에게 오류 메시지를 표시합니다

## Requirements

### Functional Requirements

- **FR-001**: 새로고침 버튼은 닫기(x) 버튼과 최소 16px 이상의 간격을 유지해야 합니다
- **FR-002**: 새로고침 버튼에 툴팁을 표시하여 기능을 명확히 안내해야 합니다
- **FR-003**: 새로고침 버튼의 크기는 Obsidian 기본 버튼 스타일과 일관되어야 합니다
- **FR-004**: 새로고침 중에는 로딩 상태를 시각적으로 표시해야 합니다
- **FR-005**: 새로고침 버튼은 직관적인 아이콘(회전 화살표)을 사용해야 합니다

### UI/UX 개선사항

- **버튼 위치**: 닫기 버튼과 충분한 간격 확보 (다른 툴바 요소들 사이에 배치 권장)
- **버튼 크기**: Obsidian CSS 변수를 활용하여 일관된 크기 적용
- **시각적 피드백**: 호버 상태, 클릭 상태, 로딩 상태를 명확히 구분
- **툴팁**: "발행 목록 새로고침" 등의 설명 텍스트 제공

## Success Criteria

### Measurable Outcomes

- **SC-001**: 사용자가 새로고침 버튼을 1초 이내에 인식하고 용도를 파악할 수 있어야 합니다
- **SC-002**: 새로고침 버튼과 닫기 버튼 간의 오클릭율이 0%에 가까워야 합니다
- **SC-003**: 버튼 크기가 Obsidian 기본 아이콘 버튼과 동일하게 유지되어야 합니다
- **SC-004**: 새로고침 버튼에 마우스를 올리면 툴팁이 표시되어야 합니다

## Assumptions

- Obsidian의 기존 CSS 변수 및 스타일 가이드라인을 따릅니다
- TailwindCSS v4를 사용하여 스타일링합니다
- 기존 발행 대시보드 Modal 구조를 유지합니다
- Lucide 아이콘 라이브러리를 사용합니다 (Obsidian 기본)
