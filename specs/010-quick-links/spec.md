# Feature Specification: GitHub 저장소 및 배포 사이트 바로가기 버튼 추가

**Feature Branch**: `010-quick-links`
**Created**: 2026-01-15
**Status**: Draft
**Input**: User description: "GitHub 저장소 및 배포 사이트 바로가기 버튼 추가"
**Linear Issue**: JEO-6

## User Scenarios & Testing *(mandatory)*

### User Story 1 - GitHub 저장소 바로가기 (Priority: P1)

사용자가 플러그인 설정에서 연결된 GitHub 저장소로 빠르게 이동하여 발행된 콘텐츠를 확인하거나 관리할 수 있습니다.

**Why this priority**: 사용자들이 가장 자주 필요로 하는 기능으로, 발행 후 콘텐츠 확인이나 문제 해결 시 GitHub 저장소 접근이 필수적입니다.

**Independent Test**: 설정 화면에서 GitHub 버튼을 클릭하여 외부 브라우저에서 저장소 페이지가 열리는지 확인할 수 있습니다.

**Acceptance Scenarios**:

1. **Given** 사용자가 GitHub 저장소를 설정한 상태에서, **When** 설정 탭 상단의 "GitHub 저장소" 버튼을 클릭하면, **Then** 외부 브라우저에서 해당 GitHub 저장소 페이지가 열립니다.
2. **Given** 사용자가 GitHub 저장소를 설정하지 않은 상태에서, **When** 설정 탭을 열면, **Then** "GitHub 저장소" 버튼이 비활성화 상태로 표시됩니다.
3. **Given** 사용자가 GitHub 저장소를 설정한 상태에서, **When** 커맨드 팔레트에서 "GitHub 저장소 열기" 명령을 실행하면, **Then** 외부 브라우저에서 해당 GitHub 저장소 페이지가 열립니다.

---

### User Story 2 - 배포 사이트(홈페이지) 바로가기 (Priority: P1)

사용자가 Quartz로 배포된 실제 웹사이트로 빠르게 이동하여 발행된 콘텐츠가 어떻게 보이는지 확인할 수 있습니다.

**Why this priority**: GitHub 저장소 바로가기와 동등하게 중요한 기능으로, 사용자들이 발행 결과를 즉시 확인하는 데 필수적입니다.

**Independent Test**: 설정 화면에서 홈페이지 버튼을 클릭하여 외부 브라우저에서 배포된 사이트가 열리는지 확인할 수 있습니다.

**Acceptance Scenarios**:

1. **Given** 사용자가 Quartz 설정에서 baseUrl을 설정한 상태에서, **When** 설정 탭 상단의 "홈페이지" 버튼을 클릭하면, **Then** 외부 브라우저에서 해당 배포 사이트가 열립니다.
2. **Given** 사용자가 baseUrl을 설정하지 않은 상태에서, **When** 설정 탭을 열면, **Then** "홈페이지" 버튼이 비활성화 상태로 표시됩니다.
3. **Given** 사용자가 baseUrl을 설정한 상태에서, **When** 커맨드 팔레트에서 "배포 사이트 열기" 명령을 실행하면, **Then** 외부 브라우저에서 해당 배포 사이트가 열립니다.

---

### User Story 3 - 커맨드 팔레트를 통한 빠른 접근 (Priority: P2)

키보드 중심의 사용자가 어디서든 커맨드 팔레트를 통해 GitHub 저장소나 배포 사이트에 빠르게 접근할 수 있습니다.

**Why this priority**: 설정 탭을 열지 않고도 빠르게 접근할 수 있는 보조 기능으로, 고급 사용자의 워크플로우를 개선합니다.

**Independent Test**: 옵시디언 어디서나 커맨드 팔레트를 열고 명령을 실행하여 해당 링크가 열리는지 확인할 수 있습니다.

**Acceptance Scenarios**:

1. **Given** 사용자가 옵시디언을 사용 중일 때, **When** 커맨드 팔레트(Ctrl/Cmd+P)를 열고 "Quartz"를 검색하면, **Then** "GitHub 저장소 열기"와 "배포 사이트 열기" 명령이 표시됩니다.

---

### Edge Cases

- GitHub 저장소 URL 형식이 유효하지 않은 경우 어떻게 처리하나요? → 버튼을 비활성화하고 설정 검증 시 오류 메시지를 표시합니다.
- baseUrl에 http/https 프로토콜이 없이 입력된 경우 어떻게 처리하나요? → https:// 프로토콜을 자동으로 추가합니다.
- 네트워크 연결이 없는 오프라인 상태에서 버튼을 클릭하면 어떻게 되나요? → 외부 브라우저로 URL을 전달하고, 브라우저가 오프라인 상태를 처리합니다.
- 설정이 변경되어 URL이 무효화된 경우 어떻게 처리하나요? → 사용자가 클릭 시점에 현재 저장된 설정 값을 사용합니다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 설정 탭 상단에 "GitHub 저장소 열기" 버튼을 표시해야 합니다.
- **FR-002**: 시스템은 설정 탭 상단에 "배포 사이트 열기" 버튼을 표시해야 합니다.
- **FR-003**: GitHub 저장소 버튼 클릭 시, 시스템은 설정된 GitHub 저장소 URL을 외부 브라우저에서 열어야 합니다.
- **FR-004**: 배포 사이트 버튼 클릭 시, 시스템은 Quartz 설정의 baseUrl을 외부 브라우저에서 열어야 합니다.
- **FR-005**: GitHub 저장소가 설정되지 않은 경우, 시스템은 해당 버튼을 비활성화 상태로 표시해야 합니다.
- **FR-006**: baseUrl이 설정되지 않은 경우, 시스템은 해당 버튼을 비활성화 상태로 표시해야 합니다.
- **FR-007**: 시스템은 커맨드 팔레트에 "GitHub 저장소 열기" 명령을 등록해야 합니다.
- **FR-008**: 시스템은 커맨드 팔레트에 "배포 사이트 열기" 명령을 등록해야 합니다.
- **FR-009**: baseUrl에 프로토콜(http/https)이 없는 경우, 시스템은 https://를 자동으로 추가해야 합니다.
- **FR-010**: 버튼은 시각적으로 현재 테마와 일관성 있게 표시되어야 합니다.

### Key Entities

- **GitHub 저장소 URL**: 플러그인 설정에서 관리되는 사용자의 GitHub 저장소 주소
- **배포 사이트 URL (baseUrl)**: Quartz 설정에서 가져오는 배포된 웹사이트의 기본 URL

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 사용자는 설정 화면에서 1번의 클릭으로 GitHub 저장소에 접근할 수 있습니다.
- **SC-002**: 사용자는 설정 화면에서 1번의 클릭으로 배포 사이트에 접근할 수 있습니다.
- **SC-003**: 사용자는 옵시디언 어디서나 커맨드 팔레트를 통해 3단계(팔레트 열기 → 검색 → 선택) 이내에 바로가기에 접근할 수 있습니다.
- **SC-004**: 설정되지 않은 바로가기의 경우, 사용자는 비활성화된 버튼을 통해 설정이 필요함을 명확히 인지할 수 있습니다.
- **SC-005**: 바로가기 버튼은 현재 옵시디언 테마(라이트/다크)에 적절히 맞춰 표시됩니다.

## Assumptions

- GitHub 저장소 URL은 기존 플러그인 설정에서 이미 관리되고 있습니다.
- baseUrl은 Quartz 설정 기능(004-advanced-quartz-config)에서 이미 구현되어 가져올 수 있습니다.
- 외부 브라우저로 URL을 여는 기능은 옵시디언 API를 통해 지원됩니다.
- 버튼 스타일링은 기존 TailwindCSS 설정과 옵시디언 CSS 변수를 활용합니다.
