# Feature Specification: GitHub 리포지토리 설정 가이드

**Feature Branch**: `011-github-guide`
**Created**: 2026-01-15
**Status**: Draft
**Input**: User description: "초보자가 깃헙 리포지토리 생성하는 방법에 대한 가이드 또는 문서 제공"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 플러그인 내 가이드 접근 (Priority: P1)

GitHub 사용 경험이 없는 초보자가 Obsidian 플러그인 내에서 Quartz 리포지토리 설정 방법에 대한 안내를 받을 수 있다.

**Why this priority**: 초보자가 플러그인 사용을 시작할 때 가장 먼저 직면하는 장벽이 GitHub 리포지토리 설정이므로, 이 단계에서 명확한 안내가 없으면 플러그인 사용 자체를 포기할 수 있다.

**Independent Test**: 설정 탭에서 가이드 버튼을 클릭하여 가이드 모달이 표시되고, 각 단계별 설명을 확인할 수 있다.

**Acceptance Scenarios**:

1. **Given** 플러그인 설정 탭에 있을 때, **When** "GitHub 설정 가이드" 버튼을 클릭하면, **Then** 단계별 설정 가이드 모달이 표시된다
2. **Given** 가이드 모달이 열려 있을 때, **When** 각 단계를 확인하면, **Then** 스크린샷/이미지와 함께 명확한 설명이 표시된다
3. **Given** 가이드를 읽고 있을 때, **When** 외부 링크(GitHub 문서 등)를 클릭하면, **Then** 기본 브라우저에서 해당 페이지가 열린다

---

### User Story 2 - Quartz 템플릿 Fork 안내 (Priority: P1)

초보자가 Quartz 공식 템플릿을 자신의 GitHub 계정으로 Fork하는 방법을 단계별로 안내받을 수 있다.

**Why this priority**: Fork는 Quartz 사용의 첫 단계이며, 이 과정 없이는 플러그인 기능을 전혀 사용할 수 없다.

**Independent Test**: 가이드의 Fork 섹션을 따라하여 실제로 Quartz 리포지토리를 Fork할 수 있다.

**Acceptance Scenarios**:

1. **Given** 가이드의 Fork 단계에 있을 때, **When** "Quartz 템플릿 열기" 버튼을 클릭하면, **Then** 브라우저에서 jackyzha0/quartz 리포지토리 페이지가 열린다
2. **Given** Fork 방법 설명을 보고 있을 때, **When** 각 단계를 확인하면, **Then** Fork 버튼 위치와 설정 방법이 스크린샷과 함께 표시된다
3. **Given** Fork 완료 후, **When** 플러그인 설정에 리포지토리 URL을 입력하면, **Then** 연결 성공 여부가 표시된다

---

### User Story 3 - GitHub Personal Access Token 생성 안내 (Priority: P1)

초보자가 플러그인 인증에 필요한 GitHub Personal Access Token(PAT)을 생성하고 설정하는 방법을 안내받을 수 있다.

**Why this priority**: PAT 없이는 플러그인이 GitHub에 접근할 수 없어 발행 기능이 동작하지 않는다.

**Independent Test**: 가이드의 PAT 생성 섹션을 따라하여 실제로 토큰을 생성하고 플러그인에 설정할 수 있다.

**Acceptance Scenarios**:

1. **Given** 가이드의 PAT 섹션에 있을 때, **When** "토큰 생성 페이지 열기" 버튼을 클릭하면, **Then** 브라우저에서 GitHub 토큰 설정 페이지가 열린다
2. **Given** PAT 생성 방법 설명을 보고 있을 때, **When** 권한 설정 단계를 확인하면, **Then** 필요한 최소 권한(repo 범위)이 명확히 표시된다
3. **Given** 토큰 생성 완료 후, **When** 플러그인 설정에 토큰을 입력하면, **Then** 토큰 유효성 검증 결과가 표시된다

---

### User Story 4 - 진행 상황 체크리스트 (Priority: P2)

초보자가 설정 과정에서 자신이 어느 단계까지 완료했는지 확인할 수 있다.

**Why this priority**: 설정 과정이 여러 단계로 나뉘어 있어, 진행 상황 추적이 없으면 사용자가 혼란을 느낄 수 있다.

**Independent Test**: 가이드 내 체크리스트를 통해 완료된 단계와 남은 단계를 확인할 수 있다.

**Acceptance Scenarios**:

1. **Given** 가이드 모달이 열려 있을 때, **When** 설정 진행 상황을 확인하면, **Then** 각 단계(GitHub 계정, Fork, PAT, 리포지토리 연결)의 완료 여부가 표시된다
2. **Given** 플러그인 설정이 완료된 상태에서, **When** 가이드를 열면, **Then** 해당 단계가 자동으로 "완료" 상태로 표시된다

---

### User Story 5 - 문제 해결 안내 (Priority: P3)

초보자가 설정 과정에서 발생할 수 있는 일반적인 문제에 대한 해결 방법을 찾을 수 있다.

**Why this priority**: 문제 해결 없이도 기본 기능은 사용 가능하지만, 오류 발생 시 사용자 지원 부담을 줄일 수 있다.

**Independent Test**: 가이드의 문제 해결 섹션에서 일반적인 오류에 대한 해결책을 확인할 수 있다.

**Acceptance Scenarios**:

1. **Given** 가이드의 문제 해결 섹션에 있을 때, **When** 일반적인 오류 목록을 확인하면, **Then** 각 오류에 대한 원인과 해결 방법이 표시된다
2. **Given** 특정 오류가 발생했을 때, **When** 해당 오류 메시지로 검색하면, **Then** 관련 해결 방법이 표시된다

---

### Edge Cases

- GitHub 계정이 없는 사용자가 가이드에 접근하면 어떻게 되는가?
  - 계정 생성 링크와 간단한 안내를 먼저 제공한다
- 사용자가 이미 Quartz 리포지토리를 가지고 있는 경우 어떻게 되는가?
  - 기존 리포지토리 연결 옵션을 안내한다
- PAT이 만료되거나 권한이 부족한 경우 어떻게 되는가?
  - 토큰 재생성 가이드로 안내한다
- 네트워크 연결이 없을 때 가이드에 접근하면 어떻게 되는가?
  - 가이드 내용은 플러그인에 내장되어 오프라인에서도 표시된다
- 영어가 익숙하지 않은 사용자가 GitHub 인터페이스를 이해하지 못하면 어떻게 되는가?
  - 가이드 내 스크린샷에 한글 설명을 오버레이로 추가한다

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 설정 탭에서 "GitHub 설정 가이드" 버튼을 제공해야 한다
- **FR-001a**: 시스템은 필수 설정(GitHub 토큰, 리포지토리)이 미완료 상태일 때 가이드를 자동으로 표시해야 한다
- **FR-002**: 시스템은 단계별 가이드를 스텝 위자드 형식의 모달로 표시해야 한다 (이전/다음 버튼, 진행바 포함)
- **FR-003**: 시스템은 Quartz 템플릿 Fork 과정을 스크린샷과 함께 안내해야 한다
- **FR-004**: 시스템은 GitHub Personal Access Token 생성 과정을 스크린샷과 함께 안내해야 한다
- **FR-005**: 시스템은 필요한 PAT 권한(repo 범위)을 명확히 표시해야 한다
- **FR-006**: 시스템은 외부 링크(GitHub 페이지)를 기본 브라우저에서 열어야 한다
- **FR-007**: 시스템은 가이드 내용을 오프라인에서도 표시할 수 있어야 한다
- **FR-008**: 시스템은 현재 설정 상태를 기반으로 진행 상황을 표시해야 한다
- **FR-009**: 시스템은 일반적인 오류에 대한 문제 해결 가이드를 제공해야 한다
- **FR-010**: 시스템은 한글로 가이드 내용을 제공해야 한다

### Key Entities

- **Guide Step**: 각 설정 단계를 나타내며, 제목/설명/스크린샷/외부링크/완료조건을 포함한다
- **Setup Status**: 사용자의 현재 설정 완료 상태를 나타내며, 각 단계별 완료 여부를 포함한다
- **Troubleshooting Item**: 문제 해결 항목을 나타내며, 오류 메시지/원인/해결 방법을 포함한다

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 초보자가 가이드만으로 10분 이내에 리포지토리 설정을 완료할 수 있다
- **SC-002**: 가이드의 각 단계가 3개 이하의 하위 작업으로 구성된다
- **SC-003**: 사용자의 80% 이상이 별도의 외부 도움 없이 초기 설정을 완료한다
- **SC-004**: 설정 관련 사용자 문의가 50% 이상 감소한다
- **SC-005**: 가이드 모달이 1초 이내에 표시된다

## Assumptions

- 사용자는 기본적인 웹 브라우저 사용 경험이 있다
- GitHub 회원가입은 별도의 외부 프로세스로 진행되며, 플러그인은 이를 링크로만 안내한다
- 스크린샷/이미지는 플러그인 빌드 시 정적 자산으로 포함되어 오프라인에서도 표시된다
- GitHub UI가 변경될 경우 스크린샷 업데이트가 필요할 수 있다
- 가이드 내용은 한국어로 제공되며, 향후 다국어 지원은 별도 기능으로 확장 가능하다
